Optimize AfuChat storage architecture to behave like Telegram/WhatsApp instead of a cache-heavy web-wrapper app.

STACK:
- React Native Expo
- Expo Router
- Supabase
- Cloudflare R2
- expo-file-system
- expo-image
- expo-sqlite or react-native-mmkv

OBJECTIVE:
Move important media from cache storage into persistent user storage so Android classifies it as “User data” instead of “Cache”.

IMPLEMENTATION REQUIREMENTS:

1. STORAGE STRUCTURE

Use:
- FileSystem.documentDirectory for permanent media
- FileSystem.cacheDirectory only for temporary files

Create persistent directories:
- /AfuChat/media/images/
- /AfuChat/media/videos/
- /AfuChat/media/audio/
- /AfuChat/media/profiles/
- /AfuChat/downloads/
- /AfuChat/temp/

2. MEDIA STORAGE RULES

Permanent storage:
- chat images
- chat videos
- voice notes
- downloaded files
- viewed media for offline access

Temporary cache:
- thumbnails
- previews
- temporary uploads
- feed preloads
- API cache

Never store permanent media in cacheDirectory.

3. MEDIA DOWNLOAD FLOW

When media is requested:

- check SQLite/MMKV for local file path
- if local file exists:
  - load local file immediately
  - skip network request
- if file missing:
  - download from Cloudflare R2
  - save to documentDirectory
  - save metadata locally

Metadata:
- remote_url
- local_path
- mime_type
- file_size
- created_at
- last_accessed
- owner_id/message_id

4. FILE NAMING

Generate deterministic hashed filenames.

Examples:
- img_a82hd72.webp
- vid_j27s82.mp4
- aud_p92hs8.m4a

Avoid random temporary names.

5. IMAGE & VIDEO OPTIMIZATION

Use expo-image:
- cache thumbnails only
- lazy loading
- progressive image loading
- prefetch visible media

Uploads:
- compress images to WebP
- compress videos before upload
- adaptive quality for weak networks

6. OFFLINE-FIRST SYSTEM

Requirements:
- viewed media opens offline
- reuse local media first
- avoid repeated downloads
- local-first lookup
- remote fallback

7. CACHE MANAGEMENT

Implement automatic cleanup:
- delete temp files older than 7 days
- remove orphaned cache
- remove duplicate temp files
- enforce cache size limit

Never auto-delete permanent user media.

8. STORAGE MANAGER

Create storage management screen:
- total images size
- videos size
- audio size
- cache size
- clear cache action

Clear cache must not delete permanent media.

9. PERFORMANCE

Optimize for:
- low-end Android devices
- low RAM
- slow networks
- low bandwidth

Avoid:
- loading large files into memory
- duplicate downloads
- excessive cache growth
- unnecessary re-renders

10. EXPECTED RESULT

Android Settings should show:
- lower Cache usage
- higher User data usage
- persistent offline media

App should behave like:
- Telegram
- WhatsApp
- TikTok offline media systems
