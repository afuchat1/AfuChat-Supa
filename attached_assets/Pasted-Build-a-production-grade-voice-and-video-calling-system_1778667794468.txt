Build a production-grade voice and video calling system for AfuChat using React Native with WebRTC.

Main goal:

- Calls must remain stable on weak mobile networks
- Audio should always be prioritized over video
- Automatic recovery/reconnection when internet fluctuates
- Low data usage and low latency
- Similar experience to WhatsApp/Telegram calls

Core stack:

- React Native
- react-native-webrtc
- Expo Dev Client (not Expo Go)
- Supabase Realtime or Socket.IO for signaling
- Firebase Cloud Messaging (FCM) for incoming calls
- Coturn TURN/STUN servers
- Internal encrypted storage for call history/settings

Required features:

1. One-to-one audio calls
2. One-to-one video calls
3. Incoming call screen
4. Ringing state
5. Accept/reject calls
6. Mute microphone
7. Toggle speaker
8. Toggle camera
9. Switch front/back camera
10. End call
11. Call duration timer
12. Missed call tracking
13. Background incoming calls
14. Auto reconnect during unstable internet
15. Audio-only fallback when bandwidth is poor

Network optimization:

- Use adaptive bitrate
- Reduce video resolution dynamically
- Reduce FPS dynamically
- Preserve audio quality first
- Automatically switch:
  - 720p → 480p → 240p → audio only
- Use Opus codec for audio
- Use VP9 or H264 for video
- Implement jitter buffering
- Implement packet loss recovery
- Implement ICE restart on connection failure

TURN/STUN:

- Configure multiple TURN servers
- Primary: Coturn VPS
- Backup TURN relay support
- Use secure TURN credentials
- Retry ICE candidates automatically

Signaling flow:

- User A starts call
- Send invite through signaling server
- Push notification wakes User B
- Exchange SDP offers/answers
- Exchange ICE candidates
- Establish WebRTC peer connection
- Monitor network quality continuously

Call reliability:

- Detect weak network in real time
- Auto reconnect without user interaction
- Resume call session after temporary disconnect
- Prevent duplicate call sessions
- Handle app background/foreground transitions safely

Performance requirements:

- Hardware acceleration enabled
- Native media processing only
- Lightweight call UI
- Minimal re-renders
- Optimize battery usage
- Optimize memory usage
- Avoid unnecessary state updates during calls

Permissions:
Request only:

- Microphone
- Camera
- Notifications
- Foreground service

DO NOT request:

- Call logs
- Phone state
- Real phone call permissions

Storage:
Store internally:

- Call history
- Missed calls
- User call settings
- Cached avatars

Do not store recordings unless explicitly enabled later.

Architecture:

- Separate signaling from media transport
- Never route media through backend server
- Backend handles only signaling/events
- Media must flow through WebRTC peer connection or TURN relay

Security:

- Secure signaling
- Encrypted peer connections
- Validate call sessions
- Prevent unauthorized joins
- Handle token expiration safely

Expected output:

- Full scalable architecture
- Folder structure
- Call service layer
- WebRTC manager
- TURN/STUN configuration
- Incoming call handling
- Background service handling
- Push notification integration
- Reconnection logic
- Network quality monitoring
- Optimized React Native implementation
- Production-ready codebase structure
