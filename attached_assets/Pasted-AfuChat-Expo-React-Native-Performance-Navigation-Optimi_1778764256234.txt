AfuChat Expo React Native Performance & Navigation Optimization

Current problems:

APK slower than web version
Delay on taps/navigation
White/black flash screen when going back
Pages reload instead of staying preserved
Laggy scrolling/transitions
Heavy rendering causing poor UX

Objective:
Optimize AfuChat Expo app to feel lightweight, instant, smooth, and stable like Telegram, TikTok, Instagram, and WhatsApp.

1. Expo Navigation Optimization

Use optimized Expo-compatible native navigation setup:

Required:

expo-router OR React Navigation with:
@react-navigation/native-stack
react-native-screens
react-native-safe-area-context

Enable:

import { enableScreens } from 'react-native-screens';

enableScreens(true);

Use:

freezeOnBlur: true

Avoid:

unmountOnBlur
resetting screen state
rebuilding pages on back navigation

Goal:

instant transitions
preserved scroll positions
no flashing/flickering
no page remounts
2. Fix White/Black Flash Screen

Causes:

screen remounting
background mismatch
WebView redraw
delayed rendering

Required fixes:

Match splash background color with app background
Preload next screens
Keep previous screens cached
Prevent blank loading containers
Optimize navigation transitions
Remove unnecessary loaders during navigation

Expo config:

{
  "expo": {
    "backgroundColor": "#000000"
  }
}

Ensure all screens use same background color.

3. Rendering Performance

Reduce unnecessary re-renders.

Required:

React.memo
useMemo
useCallback
optimized FlatList
FlashList (@shopify/flash-list)

Avoid:

global state triggering app-wide renders
full feed refreshes
nested heavy components

Optimize:

chats
feed
comments
notifications
profile pages
4. Local Storage Optimization

Avoid over-dependence on cache/network.

Use:

react-native-mmkv
expo-sqlite

Store locally:

chats
sessions
feed data
thumbnails
user settings
recently opened content

Goals:

instant reopening
offline support
fewer API requests
faster navigation
5. API & Network Optimization

Required:

request batching
pagination/infinite scrolling
background syncing
API caching
retry queues for bad internet
debounced API requests

Avoid:

refetching entire feeds
duplicate requests
loading all data at once
6. Media Optimization

Images/videos likely causing lag.

Use:

expo-image
lazy loading
thumbnail previews
compressed uploads
preload only visible videos
cached media rendering

Avoid:

loading full-resolution media immediately
auto-playing many videos at once
7. Expo Build Optimization

Enable production optimizations:

{
  "expo": {
    "jsEngine": "hermes"
  }
}

Required:

Hermes enabled
EAS Build production mode
remove unused dependencies
lazy imports/dynamic imports
asset optimization
font optimization
icon optimization

Reduce:

JS bundle size
memory usage
startup execution time
8. WebView Optimization

If using WebViews:

minimize WebView usage
convert important pages to native Expo screens
prevent WebView reloads
cache web content properly

WebViews should NOT reload during navigation.

9. Startup Optimization

App startup should load only essentials.

On launch:

Load only:

auth/session
basic UI
critical user data

Defer:

analytics
heavy API calls
notifications sync
feed prefetching
background tasks
10. Performance Monitoring

Use:

Flipper
React DevTools Profiler
Expo performance monitoring
Hermes profiling
Android Studio Profiler

Detect/fix:

dropped frames
memory leaks
JS thread blocking
unnecessary renders
slow transitions

Final target:

AfuChat Expo APK should:

feel instant on every tap
have smooth 60fps navigation
preserve pages during back navigation
eliminate white/black flash screens
open chats/feed instantly
stay fast even on low-end Android devices
use less mobile data
remain stable on poor internet
feel lightweight and native-quality across the entire app.
